# VERITY：最后一个问题

打开 `index.html`，点击“开始询问”即可游玩。网页无需联网，也不需要安装依赖。

## 操作

- 点击预设问题推动主线。
- 每个预设问题只能选择一次，使用后会从当前问题列表中移除。
- 在输入框中自由询问 Verity，部分关键词会解锁隐藏回答和线索。
- 使用右上角按钮重播开场语音或静音。
- 键盘可完成全部操作，页面也支持减少动画模式。

## 结局

游戏包含“真相”“逃离”“同化”“愤怒”和隐藏的“第一次放手”五个结局。不同的线索、信任度、异常值和最终回答会改变结果。

## 修改与同步剧情

`js/story-config.js` 是游戏与开发者编辑器共同读取的线上剧情文件。它集中保存章节、问题、回答、信任与异常变化、线索、崩坏路线、动画时间和资源路径。

开发者可以直接修改该文件，也可以打开 `editor.html` 使用可视化编辑器：

- `保存本地预览`：只影响当前浏览器，适合立即打开游戏检查修改。
- `导出线上剧情文件`：生成可直接部署的 `story-config.js`。
- `导出 JSON`：备份全部剧情配置。
- `导入 JSON`：恢复备份并继续编辑。
- `恢复线上配置`：清除当前浏览器的预览内容，重新读取线上文件。

编辑器会检查重复章节编号、重复问题 ID、无效分数和章节推进。第一至第十四章必须各有一个推进到下一章的正确选项，第十五章保留结局选项。

要让所有在线玩家同步修改，请用导出的文件替换仓库中的 `public/js/story-config.js`，提交到 GitHub，并等待 Cloudflare Workers 部署完成。公开网页不会保存 GitHub 或 Cloudflare 密钥。

角色图片与开场音频来自用户提供的本地素材。游戏剧情根据用户提供的 Verity 人物传记改编。

隐藏完美结局会播放 `assets/verity-victory.mpeg`。关键错误包含贴脸惊吓或 `ONLY ME` 强制纠正事件；愤怒结局会进入全屏红色终局动画。

页面左下角的 `DEV` 按钮可打开结局快速预览面板，一键查看五个结局。预览不会要求先完成剧情，适合开发者修改结局文本、音频和视觉效果后立即检查。

游戏启动时会先显示闪光、抖动、突发声音和跳吓警告。玩家可以在警告页启用“减少闪光与跳动效果”，随后再进入动画标题页。
